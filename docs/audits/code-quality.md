# Code quality audit – agora

Read-only audit of `the agora repo root`. No files modified.

## Executive summary

Solid architecture for a 48-hour build: clean feature-module seams (`addRouteModule`), a genuinely decoupled in-process bus, presence kept in memory per ADR-0003, and per-request permission checks enforced consistently. Type-safety discipline is mostly strong – strict TS is on, boundaries are validated with zod, and `any` is almost absent. Weakest areas are the WS feature handlers (payload cast through `as` rather than re-validated with the shared zod schemas), inconsistent HTTP error shapes across features, a few outright bugs left in by parallel-agent seams (XMPP password argument order, dev seed permission check, room.member_removed payload drift), and production defaults in `docker-compose.yml` that leak into the delivery contract. Tests lean heavily on mocked DB chains – no real Postgres-backed integration coverage for bans, invitations, unread counters, or the presence broadcast subscriber set. Ship risk is moderate: the demo path works, but XMPP auth, dev seed room permission, and the logged reset token would bite in any post-demo use.

## Patterns worth protecting

- **Feature-module auto-registration via `addRouteModule`**. Keeps `server.ts` stable across features and minimises the blast radius of parallel agents. Only the `routes/index.ts` imports need touching. Preserve this.
- **Per-request permission checks, no caching.** `canAccessRoom`, `canSendDm`, `isBannedFromRoom`, `loadDmForUser` return discriminated `{ ok: true } | { ok: false, code }`. This is exactly right; do not let anyone "optimise" with a request-scoped memo without load evidence.
- **`connectionLifecycle`.** Decouples presence, auto-subscribe, and any future WS-lifecycle feature from `ws/plugin.ts`. Excellent seam.
- **`addRouteModule` + `registerWsHandler` + in-memory `bus` = the plug-in surface.** Three small primitives, three useful extension points. Do not layer abstractions on top.
- **UUIDv7 everywhere**, used deliberately for pagination (`history.ts` compares by `id` because UUIDv7 is k-sortable). The comment that names the invariant is load-bearing – keep it.
- **Password hashing with argon2id at OWASP 2026 parameters, deterministic test of the params string.** This is the kind of test that catches a silent downgrade.
- **Presence state machine tests (registry.spec.ts).** Eight states, fake timers, assertion per transition. Use this as the template when the sweeper gets tests.
- **Orphan attachment sweeper + disk hash-path with two-level fan-out.** Clean content addressing, dedupe by hash, bytes cleanup after DB row deletion. Correct ordering.
- **Shared zod schemas in `@agora/shared`.** Single source of truth for DTOs and WS events. Reusing them on the server side (finding 5) would make this even stronger.

## Debt list (non-urgent, worth tracking)

- Replace manual payload casts in WS handlers with shared zod schemas (finding 5) – small, mechanical, high leverage.
- Unify HTTP error shape (finding 12). Single helper, gradual migration.
- Add a shared Postgres test fixture; promote at least the ban/accept/invitation paths to DB-backed integration tests (finding 17).
- Pull attachment cascade out of `bus-handlers.ts` monkey-patch into a real bus middleware slot (finding 8, 10).
- Honour or remove `openConversationIds` in the `hello` event (finding 24).
- Wire `attachmentIds` in `message.send` (finding 6) – user-visible now that the composer uploads files.
- Drop unnecessary eslint-disable comments (finding 31) now that Biome is the linter.
- Split `rooms/routes.ts` into membership / moderation / invitations when a relevant feature lands (finding 35).
- Consider a fastify-plugin-scoped rate-limit for `/api/auth/*` endpoints – `@fastify/rate-limit` is in `package.json` but never registered.
- Graceful shutdown should surface `app.close` / `pool.end` errors (finding 20).
- `config.SESSION_SECRET` should be required in production (finding 4, 32).
- Backfill `docs/ws-protocol.md` examples against the actual publisher payloads (finding 7) so the schema is not one of two sources of truth.
