# Accessibility audit ‚Äď agora web client

Date: 2026-04-17. Scope: `apps/web/src/**` (React 19 + react-router). Target: WCAG 2.1 AA, keyboard-navigable, screen-reader-usable. Read-only audit; no code changed.

## Executive summary

The design-system primitives are clean and tokenised, but accessibility has been treated as incidental rather than designed-in. Three systemic gaps dominate: (a) no visible focus styles anywhere ‚Äď every native outline is either stripped (inputs, composer) or never customised (everything else), which makes the app effectively unusable with a keyboard; (b) the modal and overlay pattern lacks `aria-modal`, focus trap, focus return, and Escape handling ‚Äď all dialogs (sign-in, register, Create room, Manage room) are affected; (c) dynamic content ‚Äď incoming messages, toasts, form errors, async statuses ‚Äď has no live-region announcement, so screen-reader users get silence. On top of this, several `<div>`s carry `onClick` (RoomListItem, ContactListItem, the modal backdrop) and a `<span>` is used for a Sign-out action; `ink-3` (#9a988c) is used for body-sized text across timestamps, hints and empty states at roughly 2.6‚Äď2.8:1 contrast, well below AA. Presence shapes are distinct and well done, and form inputs do use `useId`+`htmlFor` correctly.

## Things done well

- Form labelling. `Input.tsx` correctly uses `useId()` and associates `<label htmlFor>` with the `<input id>`. Every call-site passes `label=` and auto-complete hints (`email`, `new-password`, etc.).
- Presence cues. `Presence.tsx` uses shape (filled / half-diagonal / outlined) layered onto colour ‚Äď works for colourblind users. NFR-A11Y-1 is met where the component is used.
- Semantic-ish landmarks in `RootLayout.tsx` ‚Äď `<header>`, `<main>`, and `<aside>` for the sidebar are real HTML elements rather than `<div>`.
- The Modal close button uses `aria-label="Close"` for the `√ó` glyph.
- Contrast on the primary ink/paper pairings is fine: `ink-0` on `paper-1` is ~14:1, `ink-2` on `paper-1` is ~4.8:1, `ink-2` on white is ~5.2:1 ‚Äď all AA.
- The Composer supports Shift-Enter for newline and Enter for send; the hint is surfaced inline.
- TanStack Query already gives each mutation a loading state ‚Äď wiring `aria-busy` on triggered buttons is cheap once the convention is set.

## Quick wins

All of the below are under 10 minutes each ‚Äď good for a focused afternoon pass.

1. Add a global `:focus-visible` rule in `apps/web/src/styles/base.css` and a `.sr-only` utility. One-off CSS change, fixes findings 1 and enables 11, 18, 25.
2. Change `RoomListItem.tsx` and `ContactListItem.tsx` root `<div>` to `<button type="button">` with the existing styles reset (`background: none; text-align: left; width: 100%;`). Fixes finding 4, gets focus ring for free once (1) is done.
3. Darken `--ink-3` in `tokens.css` from `#9a988c` to approximately `#6f6d62`. Single-line token change, fixes finding 7 across ~10 call-sites.
4. Change `NavTab` to render `<button>` when it has `onClick`, else `<span>`. Fixes finding 5 and keeps call-sites untouched.
5. In `SignInPage.tsx` / `RegisterPage.tsx`, wrap the error `<div>` with `role="alert"`. Two-line change each. Fixes finding 10 for the auth path.
6. Swap `var(--accent)` to `var(--accent-ink)` on the "Forgot password?" and "Register" / "Already have an account?" links. Fixes finding 8.
7. In `RootLayout.tsx` add `aria-label="Primary"` on a new `<nav>` wrapping the NavLinks and `aria-current={active(prefix) ? 'page' : undefined}` threaded through `NavTab`. Fixes finding 17.
8. Remove the hover gate on `MessageActions` in `MessageList.tsx` ‚Äď render them inside a wrapper with `:focus-within { opacity: 1 }`, `opacity: 0` by default. Fixes finding 6.
9. Wrap the `MessageList` scroller with `role="log" aria-live="polite" aria-relevant="additions"`. Two-attribute change, fixes finding 3 at a baseline level.
10. Add `aria-modal="true"` to `Modal.tsx` and an `Escape` listener ‚Äď two attributes plus a `useEffect`. Partial fix for finding 2 while the full focus-trap lands later.
