# 2026-04-18 · Wave-3 audit closure

Short entry to close the loop on audit items that survived wave-1 and wave-2. Scope was "everything in the audit reports that's materially wrong, stop at diminishing returns" rather than "every finding gets a PR".

## What shipped

### User-visible bug — attachments never linking

`sendMessagePayload` has carried `attachmentIds` since day one of messaging; the composer + `useUpload` were wired up at some point but `message.send` on the server wasn't doing anything with the ids. Every upload stayed `messageId = NULL`, visible only to its uploader, and got swept an hour later. Industrial-grade chat with silently-orphaned uploads is not industrial-grade.

The fix is an `UPDATE` inside the send transaction, scoped tightly:

```ts
await tx
  .update(attachments)
  .set({ messageId: inserted.id })
  .where(
    and(
      inArray(attachments.id, payload.attachmentIds),
      eq(attachments.uploaderId, userId),
      isNull(attachments.messageId),
    ),
  );
```

Three gates to avoid misuse:

- **id must be in the caller's provided list** — no "catch all pending uploads" semantics.
- **uploaderId must match the caller** — you can only attach bytes you uploaded.
- **messageId must currently be NULL** — already-linked attachments can't be reassigned to a different message.

Ids that miss any gate are silently ignored; the orphan sweeper cleans up the genuinely dangling ones. No 400 on a partial match because there's no legitimate flow where a client could race its own upload cleanly, and returning 400 would surface an error the user didn't cause.

### Event-publisher / shared-schema drift

Audit code-quality finding #7 flagged two events where the publisher's payload didn't match the zod schema in `@agora/shared`:

- `room.member_removed`: publisher wrote `{ roomId, userId, by }`, schema declared `{ roomId, userId, byUserId }`.
- `invitation.received`: publisher wrote `{ id, room: {…}, inviter, createdAt }`, schema declared `{ id, roomId, roomName, inviter }`.

The web client doesn't validate with the shared zod today — it tolerates any shape via loose accessors. So this wasn't user-visible *yet*, but it's exactly the drift that bites the first time someone adds client-side validation or another consumer. Both publishers canonicalised to the schema shape; `InvitationReceivedPayload` interface follows.

The integration tests that previously "passed" on the old shape were themselves broken for a different reason: `bus.publish` pre-stringifies the event and passes `(event, serialised?)` to handlers (wave-1 perf fix), but the spec assertions used `toHaveBeenCalledWith({event})` — strict single-argument match — so every event-publisher test was quietly failing until this session, apparently uncaught. Added `, undefined` to each assertion and a new case for `room.member_removed` to lock in the canonical shape.

### Security hardening

- **Postgres host port removed** from `docker-compose.yml`. The api still reaches db on the compose network; anyone wanting psql access can `docker compose exec db psql -U app -d app` or add a `docker-compose.override.yml` locally (which is .gitignored convention).
- **Attachment mime allowlist + nosniff**. Upload path normalises `part.mimetype` against a small image allowlist (png, jpeg, gif, webp, avif) and falls back to `application/octet-stream` for everything else, capping stored mime to shapes the browser can't be tricked into rendering inline. Download response adds `X-Content-Type-Options: nosniff` so even if the stored mime were wrong the browser won't sniff around it. `originalFilename` truncated to 255 bytes before storage.

### Accessibility

Wave-1 hit the critical three (focus-visible, Modal, MessageList live region) and several highs. Wave-3 covers the remaining ones that cost under 15 minutes each:

- **Composer**: `<label>` (sr-only) + `aria-describedby` pointing to the "⏎ send · ⇧⏎ newline" hint. Screen readers now announce "Message, edit text, ⏎ send ⇧⏎ newline" instead of "edit text, blank".
- **RootLayout**: primary nav wrapped in `<nav aria-label="Primary">`; each `NavLink` threads `aria-current="page"` when active. `Sign out` now renders a real `<button>` (via a typed `as` prop on `NavTab`) instead of a `<span onClick>` — keyboard reachable, correctly announced.
- **Skip link**: visually-hidden-until-focus `Skip to content` link at the top of every page, jumping to `<main id="main">`. Saves keyboard users tabbing through 5 nav items on every navigation.
- **Auth page errors**: `role="alert"` on the error rendering in `SignInPage` and `RegisterPage`, so invalid credentials / taken username get announced instead of silently appearing.

## What was deliberately left

Diminishing returns:

- **DB-backed integration tests** for ban/accept/invitation/unread. Current specs mock the drizzle chain; a real Postgres testcontainer fixture is the right fix but is a half-day of yak-shaving plus CI plumbing that doesn't change the submission's demo behaviour. Noted on the wave-1 debt list; left there.
- **Registration enumeration** (`email_taken` vs `username_taken`). The product-side decision to surface a specific reason is defensible for UX; the enumeration cost is low. Not a critical bar.
- **`openConversationIds` on `hello`** is a dead field. Removing it means touching the shared schema, the WS plugin, and the client; low-value churn. Left as noted debt.
- **`bus.publish` monkey-patch for attachments cascade**. Audit flagged it as fragile design; agreed, but refactoring the bus to expose a wildcard `onAnyEvent` is a separate design change, not a bug fix.
- **Split `rooms/routes.ts`** into membership/moderation/invitations files. Fine as one file for now; split when a PR naturally touches only one sub-area.

## Takeaways

- **Audits earn their keep twice: once in findings, once in forcing a discipline-check.** The broken `toHaveBeenCalledWith` specs had been silently failing since wave-1's bus refactor — the rooms/friends integration files weren't in any CI gate's required path. "The audit said" isn't a fix; running the tests, reading the failures, and actually closing the loop is.
- **Contract drift between publishers and shared schemas is a time-bomb.** These schemas exist so both ends of the wire agree; publishers that don't use them regress without anyone noticing. Worth adding a publisher-side unit spec per event type that round-trips through the shared zod schema as a gate.
- **The composer attachment bug was invisible because it failed gracefully.** Uploads succeeded, UI showed a spinner, files swept up an hour later. No error, no log. That's the worst shape for a bug: everything *seems* fine. Silent correctness failures are why the "does the data actually exist in the DB after the action" check belongs in every happy-path test.
- **A11y is mostly cheap once the primitives are there.** After wave-1 set up `:focus-visible`, `.sr-only`, and Modal focus-trap, wave-3's a11y was 40 lines of attribute-plumbing. The earlier wave's investment unlocks the later wave's speed.
